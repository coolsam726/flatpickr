# RFCs

Request for Comments documents describe proposed features before they land in the package.

| RFC | Title | Status |
|-----|-------|--------|
| [0001](./0001-split-range-end-field.md) | Split range picker across two form fields | Implemented |

## Lifecycle

1. **Draft** — idea and open questions; feedback welcome.
2. **Accepted** — design approved; implementation may proceed.
3. **Implemented** — shipped; RFC is closed and linked to the merging PR.

When an RFC reaches **Implemented**, it is kept for historical context but is no longer open for design changes.
